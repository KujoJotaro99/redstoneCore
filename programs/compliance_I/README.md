# RV32I Architectural Tests

This directory contains the official `I` extension tests copied from:

```text
https://github.com/riscv/riscv-arch-test
branch: act4
path: tests/rv32i/I
commit: 7c8ed0765323d6189e627c2b94af6aff7f511e5d
```

These files are kept separate from the local `programs/rv32i` C tests so the official source tests are easy to track.

## Reference Signatures

The default compliance path uses the local RV32I Python reference model:

```sh
make -C rtl/top compliance COMPLIANCE_I="$(make -s -C programs/compliance_I list)"
```

To compare against signatures generated by the official Sail RISC-V reference model:

```sh
make -C rtl/top compliance COMPLIANCE_REFERENCE=sail COMPLIANCE_I="$(make -s -C programs/compliance_I list)"
```

From `rtl/top`, use:

```sh
make compliance COMPLIANCE_REFERENCE=sail COMPLIANCE_I="$(make -s -C ../../programs/compliance_I list)"
```

The Sail path expects `sail_riscv_sim` on `PATH`. Inside the Docker container, install the official ACT4-compatible Sail release with:

```sh
apt update
apt install -y curl ca-certificates
./programs/compliance_I/install_sail_riscv.sh /opt/sail-riscv
export PATH=/opt/sail-riscv/bin:$PATH
sail_riscv_sim --version
```

If the Sail binary has a different name or needs extra options:

```sh
make -C rtl/top compliance COMPLIANCE_REFERENCE=sail SAIL_SIM=/path/to/sail_riscv_sim SAIL_ARGS="..." COMPLIANCE_I="I-add-00"
```

The local Sail wrapper runs the model with:

```text
--rv32 --config-override=programs/compliance_I/sail_rv32_redstone_config.json --inst-limit=1000000
```

The config override matches the testbench memory map: instructions at `0x00000000` and data/signature space at `0x10000000`. The instruction limit is needed because the pure DUT harness ends with an infinite loop after writing the signature.
