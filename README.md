# RedstoneCore

  <img src="./docs/logo_dark.png" alt="RedstoneCore logo" width="720">

RedstoneCore is a 5-stage, in-order RV32I processor core.

## Status

Current checkpoint:

- 5-stage in-order pipeline: IF, ID, EX, MEM, WB
- RV32I integer compute, branch/jump, load/store, and `FENCE`
- 2-bit branch prediction with branch target buffer
- instruction prefetch FIFO
- hazard detection, load-use stalling, and forwarding
- split instruction/data cache frontends
- 4 set-associative cache with write through/write allocate, pseudo random counter eviction
- two AXI4-Lite master controllers
- shared AXI4-Lite dual-port RAM model in top-level simulation
- cocotb + Verilator verification
- official ACT4 `rv32i/I` architectural tests pass with Sail reference signatures

## Supported Instructions

RV32I computational/control/memory instruction support:

| Group | Instructions |
|---|---|
| Integer register-register | `ADD`, `SUB`, `SLL`, `SLT`, `SLTU`, `XOR`, `SRL`, `SRA`, `OR`, `AND` |
| Integer immediate | `ADDI`, `SLTI`, `SLTIU`, `XORI`, `ORI`, `ANDI`, `SLLI`, `SRLI`, `SRAI` |
| Loads | `LB`, `LH`, `LW`, `LBU`, `LHU` |
| Stores | `SB`, `SH`, `SW` |
| Branches | `BEQ`, `BNE`, `BLT`, `BGE`, `BLTU`, `BGEU` |
| Jumps | `JAL`, `JALR` |
| Upper immediates | `LUI`, `AUIPC` |
| Memory ordering | `FENCE` as in-order no-op |

## Architecture

<p align="center">
  <img src="./docs/diagram.svg" alt="RedstoneCore pipeline block diagram" width="720">
</p>

Pipeline:

- IF: PC generation, BTB prediction, instruction cache request, prefetch FIFO
- ID: decode, register file read, hazard detection, load-use scoreboard
- EX: ALU, branch/jump resolution, forwarding
- MEM: load/store formatting, data cache access, writeback source selection
- WB: register file writeback

Memory system:

- split instruction/data cache frontends
- two AXI4-Lite master controllers
- Single AXI4-Lite dual port RAM IP
- Synchronous FIFO prefetch buffering in IF

## Verification

Custom functional tests:

- arithmetic and immediates
- bitwise and shift operations
- comparisons and branch matrices
- byte/halfword/word loads and stores
- signed/unsigned load behavior
- store masking
- dependency chains
- loops and function calls
- bubble sort, binary search, prefix sum

Official architectural tests:

- ACT4 `riscv/riscv-arch-test` `rv32i/I`
- 39 instruction-focused tests
- expected signatures generated by `sail_riscv_sim`
- DUT signature memory compared in cocotb

Current result:

- all custom functional tests pass
- all 39 `COMPLIANCE_I` tests pass against Sail reference signatures

## Running Tests

Run custom regression:

```sh
cd programs/rv32i
make
```

Run official RV32I compliance regression:

```sh
cd programs/compliance_I
make
```
